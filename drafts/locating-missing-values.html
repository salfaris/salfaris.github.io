Step-by-step Exploratory Data Analysis

Posted on: Tue 13 April 2021

I think that people tend to get lost in the whole data analysis process. When I first learnt statistical modelling, I was taught to first visualize the data to get the bigger picture towards understanding the data. Usually, visualizing data would help you notice gaps in the available data almost immediately. So this strategy is not bad at all, until it does become bad. The problem is the following: in a statistical modelling class, the data you are given to work with are almost surely well-cleaned by someone already. Moreover, you might work with data that has only a single predictor (i.e. feature) for which you can almost infer some results visually. However, things are messier than this.

In reality, there is no one perfect way to do exact exploratory data analysis (EDA). This is simply because of how messy data is. In this post, my goal is to formalize a general set of rules to do EDA that is applicable to almost any problem. I will be working with Python and in particular, with pandas. We will be working with an imaginary dataset which we have put into a dataframe df. In other words, we assume that we have executed the following code:

import pandas as pd
df = pd.read_csv('some-dataset.csv')

Learning the data

Step I: head and shape

The first step that one should always do is checking the first few data by df.head(). This gives a glimpse of the content of data. The next step to do is what a lot of people usually skip until they start having problems: checking the shape of the dataframe. This can be done by simply invoking df.shape. This step might seem superfluous (especially when the data is small), but it helps you know how many rows and columns you are working with. In particular, when we start looking at missing values later on, there are sometimes odd cases where there is one whole column with missing values equal to the number of available rows! Not paying attention to this, one could end up force cleaning the data with respect to this column (e.g. by filling constant values) and use it as a predictor.
